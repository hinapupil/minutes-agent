# Minutes Agent

Discord のボイス会議を録音し、議事録とアクションアイテムを自動生成・追跡する会議アカウンタビリティ・エージェント。「会議の継続性」（前回の決定・宿題を次回に自動で持ち越す）が中核価値。

## Language

**会議 (Meeting)**:
1回の録音セッションに対応する記録単位。参加者・音声・文字起こし・議事録・アクションアイテムを束ねる。
_Avoid_: セッション、通話

**議事録 (Minutes)**:
1つの会議から生成される構造化ドキュメント（概要・決定事項・アクションアイテム・継続確認）。
_Avoid_: サマリー、要約、ノート

**アクションアイテム (Action Item)**:
議事録から抽出された、担当者を持つ宿題。open / in_progress / completed の状態を持ち、会議を跨いで追跡される。
_Avoid_: タスク、TODO

**継続確認**:
過去の会議のアクションアイテム・決定事項を新しい議事録に突き合わせるセクション。本プロダクトの差別化点。
_Avoid_: フォローアップ

**参加者ロスター (Roster)**:
会議参加者の「ユーザー ID = 表示名」の対応表。議事録生成時に注入され、話者を表示名で表記させる。

**プロジェクト用語集 (Glossary)**:
音声認識の誤変換を補正するための、ギルド単位のプロジェクト固有語彙リスト。指定された GitHub リポジトリから agent が蒸留して学習する。
_Avoid_: 辞書、静的用語集

**ギルド設定 (Guild Settings)**:
Discord サーバー（ギルド）単位の設定の集合。現在は対象リポジトリと用語集。マルチギルド展開時の拡張点（投稿先チャンネル等）。

**デモサーバー方式**:
審査員・利用者を運営側の Discord サーバーに招待して検証してもらう配布形態。録音 bot は環境依存（ボイスチャンネル・複数話者・既存データ）が強いため、ハッカソン審査の一次導線として採用。
_Avoid_: —（対比語: bot インストール型）

**bot インストール型**:
利用者が OAuth2 install link で自分のサーバーに bot を追加する標準的な配布形態。マルチギルド対応（ギルド設定の拡張）後のロードマップ。
