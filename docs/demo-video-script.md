# デモ動画 台本（2分30秒 / YouTube 限定公開）

方針: ナレーション録音はせず**字幕（テロップ）で説明**する。画面は Discord Web と LP のみ。
各シーンの「字幕」をそのまま動画エディタに貼れる形にしてある。

## シーン一覧

### S1. 課題提示（0:00–0:20）

**画面**: タイトルカード（Minutes Agent ロゴ的な1枚）→ 雑然とした会議のイメージは不要、文字だけで十分

**字幕**:
> 会議の本当の課題は「議事録を書くこと」ではない。
> **決めたことが、実行されないこと。**
> Minutes Agent は Discord の会議を録音し、議事録・宿題・進捗確認まで自律実行する AI エージェント。

### S2. エージェントがリポジトリを読んで学習（0:20–0:45）★冒頭フック

**画面**: Discord で `/setup repo:hinapupil/minutes-agent` を実行 → 「✅ 用語集を学習しました（60語）」の応答をズーム

**字幕**:
> まず `/setup` で GitHub リポジトリを指定。
> エージェントが README・docs・コントリビューターを読み、**音声認識を補正する用語集を自分で学習する。**
> 人名やプロダクト名の誤変換は、これで構造的に消える。

### S3. 録音 → 議事録（0:45–1:45）

**画面**:
1. `/join` → 「録音しますか？」ボタン → 録音開始の公開通知
2. ボイス会議の様子は数秒でカット（早送り表現可）
3. `/stop` → #議事録 に議事録が自動投稿される瞬間
4. 議事録をスクロールしながらズーム: **話者が表示名**（hina / MIYA_IF）、**Proto Pedia / MiyaIF が正しい表記**、**継続確認**（前回の決定事項との突き合わせ）セクション

**字幕**（順に）:
> `/join` で録音開始。同意は Yes/No ボタンで、開始・終了は全員に公開通知。
> `/stop` すると… 数分で議事録が自動投稿される。
> 話者は表示名で記録。学習した用語集で「プロトペディア」「みやいふ」も正しく補正。
> そして**継続確認** — 前回の会議で決めたことと突き合わせ、進捗を自動で追跡する。

### S4. 過去の議事録に質問（1:45–2:05）

**画面**: `/ask question: 次回のリリースはいつ？` → 「次回のリリースは金曜日に予定されています。」

**字幕**:
> `/ask` で過去の議事録に自然言語で質問。
> ADK エージェントが Firestore の議事録を横断検索して回答する。

### S5. アクションアイテムと自律リマインド（2:05–2:20）

**画面**: `/actions` の一覧表示

**字幕**:
> 宿題は `/actions` でいつでも確認。
> 毎朝 10 時、エージェントが期限を自律判定してリマインド — **会議と会議の間も働き続ける。**

### S6. アーキテクチャ + 締め（2:20–2:40）

**画面**: LP（稼働中バッジ→構成セクション）→ GitHub リポジトリを一瞬

**字幕**:
> すべて Google Cloud: GCE / Cloud Run / Cloud Tasks / Speech-to-Text / Gemini / ADK / Firestore。
> IaC は Terraform、デプロイは GitHub Actions（WIF keyless）。
> **Minutes Agent — 会議を、実行に変える。**

## 撮影・編集の分担（2026-07-12 調査結果）

- **撮影 = 人**: CursorFly（Chrome 拡張、タブ録画・クリック連動自動ズーム・無料 OSS）でシーンごとに分割録画。拡張が使えなければ Cmd-Shift-5（macOS 標準）。CLI 録画（screencapture/ffmpeg）は TCC 画面収録権限の壁でエージェントからは不可（実機検証済み）
- **編集 = エージェント**: 正規化 → タイトルカード生成 → concat → 字幕（SRT）焼き込み → BGM。ffmpeg で全自動、画面収録権限不要
- **YouTube アップ = 人**: 限定公開、タイトル・説明（GitHub / デプロイ URL）記入

## 撮影メモ

- 撮影前: #議事録 のデバッグ投稿掃除、`/actions` はテスト重複データが多いので見せ方注意（S5 は上位数件が映る程度でよい）
- S3 の録音会議は 30〜60 秒の短い台本会話でよい: 「次回リリースは金曜にします」「プロトペディアの公開は hina さん担当で」「みやいふさんは動画編集をお願いします」→ 議事録に用語補正・担当者・継続確認が全部乗る
- 各コマンド実行の直後 1〜2 秒静止（応答をズームで見せる余白）
- 解像度: ブラウザウィンドウを 1280×800 程度に絞ると文字が読みやすい
